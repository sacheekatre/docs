Closd's document library gives your entire deal team a single, secure place to store, organise, and share every file in the transaction. Rather than hunting through email threads and shared drives, everyone works from the same source of truth — with version history, granular access controls, and a clear folder structure that mirrors the way legal teams actually work.

## Uploading Documents

Getting documents into Closd is straightforward. Navigate to the **Documents** tab in your project, then either:

- **Drag and drop** files directly onto the document library, or
- Click the **Upload** button and select files from your computer.

Closd supports the most common legal document formats, including **PDF, Word (.docx), Excel (.xlsx), and PowerPoint (.pptx)**. The maximum file size is **100 MB per file**. For larger files — such as data room archives — consider splitting them into smaller packages before uploading.

<Tip>
  Use descriptive, dated filenames so that the right version is always obvious at a glance. A name like `SPA_v3_2024-03-15.pdf` is far clearer than `SPA_final_FINAL_v2.pdf`, and it sorts predictably in chronological order.
</Tip>

## Folder Structure

A well-organised folder structure is the foundation of an efficient deal. Closd lets you create **folders and subfolders** to group documents by category, workstream, or party — whatever makes sense for your transaction.

To create a folder, click **+ New Folder**, enter a name, and confirm. To create a subfolder, right-click an existing folder (or use its **⋮** menu) and select **New Subfolder**.

A common numbering convention for M&A transactions looks like this:

```
01 – Corporate Documents
02 – Financial Statements
03 – Contracts and Agreements
04 – Intellectual Property
05 – Real Estate
06 – Employment and Benefits
07 – Regulatory and Compliance
08 – Signed Documents
```

Numbering your folders ensures they appear in a consistent order for all users, regardless of how their browser sorts files alphabetically.

## Version History

When you upload a new version of a document that already exists in the library, Closd retains all previous versions rather than overwriting them. This gives you a complete, automatically maintained version history for every file.

To view or download an earlier version:

<Steps>
  <Step title="Locate the Document">
    Find the document in the folder structure and click on its name to open the document detail panel.
  </Step>
  <Step title="Open Version History">
    Click the **Version History** tab in the detail panel. All previous versions are listed with their upload date, file size, and the name of the user who uploaded them.
  </Step>
  <Step title="View or Download">
    Click **View** to open a version in the browser, or **Download** to save it locally. The current version is always shown at the top of the list.
  </Step>
</Steps>

You can also upload a new version directly from the document detail panel by clicking **Upload New Version**, which keeps all versions linked under a single document entry rather than creating a duplicate file.

## Sharing Documents

By default, all documents in the project are visible to every project member. This works well for open collaboration, but many transactions require tighter controls — for example, keeping sensitive financial data visible only to certain advisors.

To restrict or customise access, use Closd's [Access Rights](/projects/access-rights) settings, which let you set permissions at the folder level for individual users or groups. You can grant or deny view, download, and edit rights independently for each folder.

For a full walkthrough of how permissions work, see [Control Document and Project Access Rights](/projects/access-rights).

## Downloading Documents

To download a single document, click the **download icon** (↓) next to its name in the document library. The file will be saved to your computer in its current version.

To download an entire folder — for example, to create a local backup of signed documents at closing — right-click the folder (or use its **⋮** menu) and select **Download Folder**. Closd packages all files within the folder into a single ZIP archive for you.

<Warning>
  Documents deleted from a project are **permanently removed** and cannot be recovered. Before deleting any file, confirm that you no longer need it and that no other team member is relying on it. If in doubt, move the document to an archive folder rather than deleting it outright.
</Warning>
